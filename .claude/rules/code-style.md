> **This is a template.** Replace the placeholders with your project's

## General principles

- Prefer explicit code over clever abstractions
- Keep imports organized; remove unused ones
- Consistent naming across the codebase beats any one convention
- Format on save; never commit unformatted code
- Run the project's check command before every commit (see
  `.claude/commands/check.md`)

## Enforcement

- Run `/check` (or `make check` / `npm run check`) before every commit
- Pre-commit hooks are the safety net, not the primary check
- Never bypass hooks with `--no-verify` — fix the underlying issue
